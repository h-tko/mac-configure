# 標準開発フロー（dev-workflow）

コード変更を伴うタスクは、以下のフェーズを**順番に**守る。各フェーズの開始時に TodoWrite へ項目を起こし、完了したら次へ進む。**承認ゲート（Phase 0）を飛ばして実装に入らない。**

> **プロジェクト規約が優先**。base ブランチ名・commit 粒度・受け入れ条件の在処・RED 後の停止有無などは、対象リポジトリの `CLAUDE.md` の定義に従う。本スキルはどのプロジェクトでも変わらない骨格を定義する。

## Phase 0 — 着手前（承認ゲート）🚦

1. **要件理解**: 対応する Issue / 要件を読む。受け入れ条件・影響範囲を把握する。不明点・複数解釈は**推測で進めず質問する**
2. **ブランチ提案**: `feature/<概要>` または `fix/<概要>` を提案。**ユーザーの許可を得てから**ブランチを切る（勝手に作らない）。base ブランチはプロジェクト規約に従う
3. **実装計画の提示**: 変更対象ファイル・影響範囲・テスト方針を提示し、**ユーザーの承認を得てから**実装に入る

> Phase 0 の2つの承認（ブランチ / 計画）が揃うまで実装コードを書かない。

## Phase 1 — TDD で実装

`~/.claude/skills/tdd`（t-wada 方式）に従う。プロジェクト固有の実装パターンは各 `tdd-*` スキル / CLAUDE.md を参照。

1. **テストリスト導出** — 受け入れ条件・エッジケースから
2. **Red** — 最小のテストを1つ書き、**失敗を確認**する
3. **Green** — 仮実装で最速で通す
4. **Refactor** — 通したまま整理する
5. テストと実装の **commit 分離** や **RED 後にユーザー承認を待つ** といった運用は、プロジェクト規約に従う
6. **本番コードにテスト用分岐を入れない**（`~/.claude/rules/no-test-code-in-production.md`）

## Phase 2 — PR 作成

1. テストが書けた時点で **PR を作成**する（Draft 可）。プロジェクト規約に Issue 紐付け（`Closes #NN`）がある場合は従う
2. 実装完了後、PR 本文に **Summary / 変更内容 / Test plan** を追記する

## Phase 3 — 検証（証拠ベース）✅

`~/.claude/rules/verification-before-completion.md` に従い、**主張の前にコマンドを実行し出力で裏取り**する。

1. ユニット / 統合テストを実行しグリーンを確認
2. **E2E を実行しグリーンを確認**（`~/.claude/rules/e2e-must-pass.md`）。E2E が無い / 該当しない場合は最上位テストで代替し明示
3. lint / typecheck / build を実行
4. 結果（pass 件数等）を PR / 報告に**証拠として添える**

## Phase 4 — 自己レビュー（指摘ゼロまで）🔁

`~/.claude/rules/self-review-before-merge.md` に従う。

1. 自分の diff を**通しでレビュー**する（観点: 仕様整合 / バグ / テストカバレッジ / セキュリティ / 不要変更の混入）
2. レビュー手段は `/review`・`/code-review`・`/codex` 等を活用してよい
3. 指摘を修正し、修正を検証する（Phase 3 に戻る）
4. **新たな指摘が出なくなるまで** 1〜3 を反復する

## 禁止

- Phase 0 の承認を得ずに実装を始める
- Red を確認せずに実装する / テストリスト無しで始める
- 検証していないのに「完了」「動く」と報告する
- E2E / テストを骨抜きにして緑に見せる
- 自己レビューをせずにマージ / レビュー依頼へ進む
- 依頼にトレースできない無関係な変更を混ぜる（外科的変更）

## 全体チェックリスト（TodoWrite 化推奨）

- [ ] Phase 0: 要件理解・不明点を質問
- [ ] Phase 0: ブランチ承認を得た
- [ ] Phase 0: 実装計画の承認を得た
- [ ] Phase 1: テストリスト → Red → Green → Refactor
- [ ] Phase 2: PR 作成（Draft 可）＋本文（Summary/変更/Test plan）
- [ ] Phase 3: ユニット/統合 + E2E グリーン（証拠添付）
- [ ] Phase 4: 自己レビュー → 指摘ゼロまでループ
