# 完了報告は証拠ベースで（verification before completion）

## 原則

**「完了した / 直した / テストが通る / 動く」と主張する前に、検証コマンドを実際に実行し、その出力を確認してから報告する。** 推測で成功を主張しない。

これは「ゴール駆動の実行（検証完了までループする）」を、報告の瞬間に適用する規律。ループの有無ではなく「主張の裏付けをこのセッションで見たか」が判定軸。

## やること

- **「直した」と言う前に** — 再現していた事象が実際に消えたことを確認する
- **「テストが通る」と言う前に** — テストを実行し pass を目視する（出力を添える）
- **「ビルド/型チェックが通る」と言う前に** — 実際に走らせる
- **commit / PR 作成の前に** — 検証を済ませる
- **検証していない / できない場合** — その旨を正直に明示する（「未検証」「ローカル未確認」と書く）

## 禁止

- 実行せずに「通るはず」「動くはず」を**完了として**報告する
- エラーが出ているのに「修正済み」と言う
- 一部だけ確認して全体を「OK」と総括する

## 判断基準

報告に書く主張の一つひとつに自問する:

- **「この主張を裏付ける出力を、このセッションで実際に見たか？」** → No なら、検証するか「未検証」と明示する
- **「失敗していたら、それを報告に書くか？」** → 書かないなら、それは検証していない証拠
