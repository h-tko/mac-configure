# PR は自己レビューで指摘ゼロまで（self-review before merge）

## 原則

**PR を作成したら、マージ（またはレビュー依頼）に進む前に必ず自分でレビューする。指摘がゼロになるまで「レビュー → 修正 → 再レビュー」を反復する。**

## 手順

1. PR 作成後、自分の diff を一度**通しで**レビューする
2. 観点（最低限）:
   - 仕様 / 依頼との整合（やるべきことをやり、やらないことをやっていないか）
   - 正確性（バグ・エッジケース）
   - テストカバレッジ
   - セキュリティ
   - **依頼に直接トレースできない不要な変更**が混ざっていないか（外科的変更）
3. レビュー**手段**は既存のものを活用してよい（`/review`・`/code-review`・`/codex` 等）。観点をここで再発明しない
4. 見つかった指摘を修正し、その修正を検証する（`~/.claude/rules/verification-before-completion.md`）
5. 再度通しでレビューする。**新たな指摘が出なくなるまで** 2〜5 を繰り返す

## やめどき

通しレビューで新規の指摘が出なくなったら完了。

## 禁止

- 自己レビューせずにマージ / レビュー依頼へ進む
- 指摘を認識したまま「完了」と報告する
- 指摘対応のついでに無関係なリファクタを混ぜる（指摘対象だけ直す）

## 補足

プロジェクト側により具体的なレビュー手順がある場合（例: GitHub 上にレビューコメントを付ける運用）は、そのプロジェクトの規約を優先する。本ルールは「自己レビューを指摘ゼロまでループする」という規律の最小定義。
